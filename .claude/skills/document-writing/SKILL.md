# Document Writing

Write technical documents following Butter's established style.

## Style rules

1. **Lead with Why** — first section explains why this doc exists
2. **TL;DR at top** — 3-5 bullet summary before detail
3. **Tables for structured data** — prefer over bullet lists when 2+ dimensions
4. **Code blocks for flows** — step sequences, not prose
5. **1-line responsibility lead** — each section opens with bold one-liner
6. **No fluff** — no filler, straight to content
7. **Updated date** — `updated: YYYY-MM-DD` in frontmatter
8. **Bilingual OK** — English or Vietnamese, match context

## Anti-patterns

- Wall of prose without structure
- Bullet lists where table fits better
- Repeating title in first sentence
- Missing "Why" — jumping into details
